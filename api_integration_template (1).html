Интеграция API

Документация по подключению внешних сайтов к CRM.

Принцип работы API

API позволяет вашему сайту «общаться» с CRM через стандартные HTTP-запросы.

Основные шаги интеграции:

  • Подключение: Используйте ваш личный API-ключ в заголовке.
  • Динамика: Получение категорий для автоматического обновления формы.
  • Передача данных: Автоматическое создание лидов или тикетов.

1. Аутентификация

Все запросы должны содержать ваш личный API-ключ. Добавьте его в заголовок вашего HTTP-запроса:

x-api-key: ВАШ_API_KEY

2. Умные формы (Лиды)

Отправляйте лиды из ваших воронок продаж напрямую в CRM.

POST /inquiries

{ "title": "Новый запрос с сайта", "category_id": 1, "contact_name": "Иван Иванов", "contact_email": "ivan@example.com", "source_website": "vash-sayt.ru", "answers": [ { "question_id": 1, "answer_value": "Да" } ] }

* source_website: Опциональный параметр для отслеживания источника заявки. Если не указан, IP отправителя всё равно будет сохранен системой.

3. Поддержка (Тикеты)

Позвольте вашим клиентам создавать тикеты поддержки напрямую.

POST /support

{ "subject": "Проблема с отоплением", "description": "Отопление на 2-м этаже больше не греет. Просим проверить.", "client_name": "Анна Смирнова", "client_email": "anna@example.com" }

4. Получение категорий

Получение списка категорий, подкатегорий и вопросов для динамических форм напрямую из CRM.

GET /categories
